# E. She knows... 解题思路

### 核心问题分析 

题目的要求是：在一个 $n \times m$ 的网格中，部分格子已经被涂成了黑色（1）或白色（0）。我们需要把剩下所有绿色的格子也涂成黑色或白色，最终使得**相邻且颜色不同的格子对数**为**偶数**。求涂色的方案数。

这种要求某个全局统计量为“偶数”的题目，绝对不要去傻傻地考虑具体的涂色排列组合，它的背后一定隐藏着关于**奇偶性（Parity）**的极其优美的数学结论。

### 1. 揭开迷雾：奇偶性的本质

我们不妨用图论的视角来看待这个网格。每一个格子是一个节点 $v$，相邻的格子之间有一条边。设格子的颜色为 $c_v \in \{0, 1\}$。

如果一条边连接的两个节点 $u$ 和 $v$ 颜色不同，它对答案的贡献就是 $1$。在奇偶性的视角下（即对 $2$ 取模），颜色不同可以用异或或者加法来表示：$(c_u + c_v) \pmod 2$。

那么，整张网格中颜色不同的相邻对数 $P$，它的奇偶性为：
$$ P \equiv \sum_{\text{all edges}} (c_u + c_v) \pmod 2 $$

我们将这个“按边求和”的式子，转化为“按节点求和”。每条边会把它的两个端点各计算一次，因此对于任意一个节点 $v$，如果它的度数（相邻格子的数量）为 $d_v$，那么它的颜色 $c_v$ 在总和中就会被加 $d_v$ 次！
式子瞬间华丽变身为：
$$ P \equiv \sum_{\text{all vertices}} (d_v \times c_v) \pmod 2 $$

### 2. 破局点：谁在影响大局？

既然总对数的奇偶性取决于 $d_v \times c_v \pmod 2$，我们来看看网格中各类格子的度数 $d_v$：

1. **内部格子**：上下左右都有邻居，$d_v = 4$（偶数）。
2. **四个角落的格子**：只有两个邻居，$d_v = 2$（偶数）。
3. **边缘但非角落的格子**：有三个邻居，$d_v = 3$（奇数）。

奇妙的化学反应出现了！
对于内部格子和角落格子，它们的度数是偶数。无论涂什么颜色（$c_v = 0$ 或 $1$），$d_v \times c_v \pmod 2$ 永远是 $0$！**这就意味着，内部格子和角落格子的颜色，对最终不同颜色对数的奇偶性没有任何影响！**

对于边缘且非角落的格子，$d_v = 3$，所以 $3 \times c_v \equiv c_v \pmod 2$。
**顿悟时刻：全场唯一能决定最终奇偶性的，只有那些“边缘且非角落”的格子！**
为了让 $P$ 为偶数，这些“边缘且非角落”的格子中，黑色的数量必须是**偶数**！

### 3. 组合数学收尾

理清了条件后，方案数的计算简直轻而易举。
设全场剩下的未涂色绿格子总数为 $G$。
设“边缘且非角落”的格子中，还未涂色的绿格子数量为 $GS$。
设初始状态下，已经在“边缘且非角落”的格子中涂黑的数量为 $b$。

分两种情况讨论：

* **情况一：$GS > 0$ （边缘上还有绿格子）**
  这意味着在影响奇偶性的区域里，我们还有自由发挥的空间。我们可以把前 $GS - 1$ 个边缘绿格子随便涂，然后**最后 1 个边缘绿格子用来“背锅”**（强制调整使得黑色总数为偶数，它的颜色是唯一确定的）。
  剩下的 $G - GS$ 个非边缘绿格子，由于根本不影响奇偶性，完全可以随便涂（每个格子 $2$ 种选择）。
  所以，总方案数为：$2^{GS - 1} \times 2^{G - GS} = 2^{G - 1}$。

* **情况二：$GS = 0$ （边缘上的格子全部被涂满了）**
  这意味着奇偶性已经盖棺定论，无法改变了。
  如果初始边缘的黑格子数 $b$ 是奇数，那无论怎么涂内部的格子，奇偶性都不满足，方案数为 $0$。
  如果 $b$ 是偶数，条件天然满足，全场剩下的 $G$ 个内部绿格子随便涂，方案数为 $2^G$。

### CPP 代码实现

```cpp
// E. She knows...

#include <bits/stdc++.h>
#define lg(x) (63 - __builtin_clzll(x))
#define all(x) (x).begin(), (x).end()
#define low_bit(x) ((x) & (-x))
#define pb push_back
#define db long double
#define int long long
#define sz(x) (int)x.size()
#define endl "\n"

using namespace std;

const int MOD = 1e9 + 7;

// 快速幂模板
int qpow(int a, int b) {
    int res = 1;
    a %= MOD;
    while (b > 0) {
        if (b & 1) res = res * a % MOD;
        a = a * a % MOD;
        b >>= 1;
    }
    return res;
}

void solve() {
    int n, m, k;
    cin >> n >> m >> k;

    int a = 0; // a: 初始已经被涂色的“边缘且非角落”的格子数量
    int b = 0; // b: 初始已经被涂成黑色 (1) 的“边缘且非角落”的格子数量

    for (int i = 0; i < k; i++) {
        int x, y, c;
        cin >> x >> y >> c;
        
        // 判断当前格子是否是四个角
        bool is_corner = (x == 1 || x == n) && (y == 1 || y == m);
        // 判断当前格子是否在边缘
        bool is_edge = (x == 1 || x == n || y == 1 || y == m);
        
        // 我们只关心“边缘且非角落”的格子
        if (is_edge && !is_corner) {
            a++;
            b += c;
        }
    }
    
    // d: 全图“边缘且非角落”格子的总数
    // 上下两行各 m-2 个，左右两列各 n-2 个，加起来就是 2n + 2m - 8
    int d = 2 * n + 2 * m - 8;
    
    // GS: 还没被涂色的“边缘且非角落”的格子数量 (Green on Side)
    int GS = d - a;
    
    // G: 全图还没被涂色的格子总数
    int G = n * m - k;
    
    // 指数要对 MOD-1 取模（费马小定理），不过本题指数不大，直接用也没事
    int MOD_EXP = MOD - 1;
    
    // 分类讨论
    if (GS > 0) {
        // 如果边缘还有绿格子，其中一个必定作为“奇偶校验位”失去自由度
        // 其他所有 G - 1 个绿格子随便涂
        int exp = (G - 1) % MOD_EXP;
        cout << qpow(2, exp) << endl;
    } else {
        // 如果边缘没有绿格子了，奇偶性已被锁死
        if (b % 2 != 0) {
            // 被锁死在了奇数，神仙难救
            cout << 0 << endl;
        } else {
            // 被锁死在了偶数，天然合法，所有剩下的绿格子随便涂
            int exp = G % MOD_EXP;
            cout << qpow(2, exp) << endl;
        }
    }
}

signed main() {
    // 优化输入输出
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);

    int t = 1;
    cin >> t;

    while (t--) {
        solve();
    }
}
```
